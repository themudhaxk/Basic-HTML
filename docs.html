CITI Service API Documentation

This document provides comprehensive details about the Service Marketplace API endpoints, including request parameters, required headers, and response formats.

Base URL

plaintext http://localhost:5000

Authentication

Most endpoints require authentication using JWT tokens.

Authentication Header Format:

plaintext Authorization: Bearer {token}

Where {token} is the JWT token received from the login or refresh token endpoints.

1. Authentication

Register User

Endpoint: POST /api/auth/register

Request Body:

json { "email": string, // Required: Valid email address "password": string, // Required: Min 8 chars, 1 uppercase, 1 lowercase, 1 number "fullName": string, // Required: User's full name "phone": string, // Optional: Phone number (format: +1234567890) "role": string, // Optional: "client" (default), "vendor", "admin" "referralCode": string // Optional: Referral code from existing user }

Response (201):

json { "success": true, "message": "User registered successfully. Please check your email to verify your account.", "data": { "_id": string, "email": string, "fullName": string, "role": string, "emailVerified": boolean, "token": string, "refreshToken": string } }

Login

Endpoint: POST /api/auth/login

Request Body:

json { "email": string, // Required: User's email "password": string // Required: User's password }

Response (200):

json { "success": true, "message": "Login successful", "data": { "_id": string, "email": string, "fullName": string, "role": string, "token": string, "refreshToken": string } }

Refresh Token

Endpoint: POST /api/auth/refresh-token

Request Body:

json { "refreshToken": string // Required: Refresh token from login/register }

Response (200):

json { "success": true, "message": "Token refreshed successfully", "data": { "token": string, "refreshToken": string } }

Verify Email

Endpoint: GET /api/auth/verify-email/:token

URL Parameters:

  • token: Email verification token sent to user's email

Response (200):

json { "success": true, "message": "Email verified successfully" }

Forgot Password

Endpoint: POST /api/auth/forgot-password

Request Body:

json { "email": string // Required: User's registered email }

Response (200):

json { "success": true, "message": "Password reset email sent" }

Reset Password

Endpoint: PUT /api/auth/reset-password/:token

URL Parameters:

  • token: Password reset token from email

Request Body:

json { "password": string // Required: New password (min 8 chars, 1 uppercase, 1 lowercase, 1 number) }

Response (200):

json { "success": true, "message": "Password reset successful" }

Logout

Endpoint: POST /api/auth/logout

Headers:

  • Authorization: Bearer token

Response (200):

json { "success": true, "message": "Logged out successfully" }

OAuth Routes

  • Google Login: GET /api/auth/google
  • Google Callback: GET /api/auth/google/callback
  • Apple Login: GET /api/auth/apple
  • Apple Callback: GET /api/auth/apple/callback
  • Microsoft Login: GET /api/auth/microsoft
  • Microsoft Callback: GET /api/auth/microsoft/callback

2. Users

Get User Profile

Endpoint: GET /api/users/profile

Headers:

  • Authorization: Bearer token

Response (200):

json { "success": true, "data": { "user": { "_id": string, "email": string, "fullName": string, "phone": string, "photoUrl": string, "role": string, "emailVerified": boolean, "createdAt": string, "updatedAt": string }, "vendorProfile": object | null // Present if user is a vendor } }

Update User Profile

Endpoint: PUT /api/users/profile

Headers:

  • Authorization: Bearer token
  • Content-Type: multipart/form-data (if uploading photo)

Request Body:

plaintext fullName: string // Optional: User's full name phone: string // Optional: User's phone number photo: File // Optional: Profile photo file

Response (200):

json { "success": true, "message": "Profile updated successfully", "data": { "_id": string, "email": string, "fullName": string, "phone": string, "photoUrl": string, "role": string } }

Update User KYC Information

Endpoint: PUT /api/users/kyc

Headers:

  • Authorization: Bearer token

Request Body:

json { "bvn": string, // Optional: Bank Verification Number (Nigeria) "nin": string // Optional: National Identification Number (Nigeria) }

Response (200):

json { "success": true, "message": "KYC information updated successfully" }

Change Password

Endpoint: PUT /api/users/change-password

Headers:

  • Authorization: Bearer token

json { "currentPassword": string, // Required: Current password "newPassword": string // Required: New password (min 8 chars, 1 uppercase, 1 lowercase, 1 number) }

Response (200):

json { "success": true, "message": "Password changed successfully" }

Get User Referrals

Endpoint: GET /api/users/referrals

Headers:

  • Authorization: Bearer token

Response (200):

json { "success": true, "count": number, "data": [ { "_id": string, "referrerId": string, "referredId": { "_id": string, "fullName": string, "email": string }, "rewardGranted": boolean, "createdAt": string, "updatedAt": string } ] }

Delete User Account

  • Authorization: Bearer token

Response (200):

json { "success": true, "message": "Account deleted successfully" }

  • Authorization: Bearer token (admin only)

URL Parameters:

  • id: User ID

Request Body:

json { "status": string // Required: "active", "suspended", or "banned" }

Response (200):

json { "success": true, "message": "User status updated to {status}", "data": { "_id": string, "email": string, "fullName": string, "role": string, "accountStatus": string } }

Verify Vendor

Endpoint: PUT /api/admin/vendors/:id/verify

Headers:

  • Authorization: Bearer token (admin only)

URL Parameters:

  • id: Vendor ID

Response (200):

json { "success": true, "message": "Vendor verified successfully", "data": { "_id": string, "userId": string, "businessName": string, "verified": true, "createdAt": string, "updatedAt": string } }

Get Pending Vendors

Endpoint: GET /api/admin/vendors/pending

Headers:

  • Authorization: Bearer token (admin only)

Query Parameters:

  • page: Page number (default: 1)
  • limit: Results per page (default: 10)

Response (200):

json { "success": true, "count": number, "total": number, "pagination": { "page": number, "limit": number, "pages": number }, "data": [ { "_id": string, "userId": string, "businessName": string, "bio": string, "logoUrl": string, "certificationUrls": string[], "verified": false, "createdAt": string, "updatedAt": string, "user": { "fullName": string, "email": string, "photoUrl": string } } ] }

Release Funds from Escrow

Endpoint: PUT /api/admin/transactions/:id/release

Headers:

  • Authorization: Bearer token (admin only)

URL Parameters:

  • id: Transaction ID

Response (200):

json { "success": true, "message": "Funds released successfully", "data": { "_id": string, "bookingId": string, "amount": number, "method": string, "escrowHeld": false, "releasedToVendor": true, "reference": string, "createdAt": string, "updatedAt": string } }

Get All Transactions

Endpoint: GET /api/admin/transactions

Headers:

  • Authorization: Bearer token (admin only)

Query Parameters:

  • method: Filter by payment method
  • escrowHeld: Filter by escrow status (true/false)
  • releasedToVendor: Filter by release status (true/false)
  • page: Page number (default: 1)
  • limit: Results per page (default: 10)

Response (200):

json { "success": true, "count": number, "total": number, "pagination": { "page": number, "limit": number, "pages": number }, "data": [ { "_id": string, "bookingId": string, "amount": number, "method": string, "escrowHeld": boolean, "releasedToVendor": boolean, "reference": string, "createdAt": string, "updatedAt": string, "booking": { "status": string, "date": string, "time": string }, "client": { "fullName": string, "email": string }, "vendor": { "businessName": string, "user": { "fullName": string, "email": string } } } ] }

Error Responses

All API endpoints return standardized error responses:

400 Bad Request:

json { "success": false, "message": "Error message describing the issue", "errors": ["Detailed error information"] // Optional }

401 Unauthorized:

json { "success": false, "message": "Not authorized, token failed" }

403 Forbidden:

json { "success": false, "message": "Not authorized to access this resource" }

404 Not Found:

json { "success": false, "message": "Resource not found" }

500 Server Error:

json { "success": false, "message": "Internal Server Error" }

Rate Limiting

The API implements rate limiting to prevent abuse:

  • General endpoints: 100 requests per 15 minutes
  • Authentication endpoints: 5 requests per 15 minutes

When rate limits are exceeded, the API returns:

json { "success": false, "message": "Too many requests from this IP, please try again later." }
